Algoritmer

En algoritm är en formel för hur något ska fungera. Man kan använda en algoritm för att få hjälp till att felsöka något, med hjälp av flödesscheman kan man lättare få en grafisk presentation av ett problem. Exempelvis om inte ditt nätverk fungerar, kan första steget vara att kolla om man har kopplat alla kablar rätt, har man nu kontrollerat att alla kablar är rätt kopplade eller om man nu korrigerat problemet kan man gå vidare till nästa steg och så vidare. Inom datavetenskapen används algoritmer ofta för att lösa enkla uppgifter snabbt. En algoritm kan också beskrivas som en systematisk procedur som genom ett få antal steg utför en beräkning eller löser ett problem. En algoritm kan bestå av flera små algoritmer för att skapa en stor komplex algoritm som kan utföra stora beräkningar. De flesta algoritmer kan skrivas in i ett datorprogram och köras av datorn, men i få fall kan de bara stimuleras. Programspråk burkar kalla algoritmer för funktioner eller metoder.[1]

Programmeringsspråk

Programmeringsspråk är en kategori som innehåller flera sorters språk som exempelvis: Python, Javascript, Visual Basic, PHP & C# men det finns flera till för att läsa alla tryck här. Programspråk används för att programmera något att göra som en vill. Datorn förstår dock inte de flesta programmeringsspråk direkt utan måste först maskinkod som datorn förstår sig på. Man kan skriva i maskinkod så datorn förstår sig på språket direkt men det förekommer inte alls så ofta längre då det är alldeles för arbetskrävande flr att få fram koden. Programmering sker i ett särskilt språk i ett passande program. Det finns olika spår inom programmering exempelvis strukturerad programmering, funktionell programmering, objektorienterad programmering & logikprogrammering. Programmering kan ske i olika abstraktionsnivåer, maskinkod är den som är närmast hårdvaran. På nästa nivå kommer den nivå där det används exempelvis: Java, C & Pascal. Ada Lovelace brukar benämnas som den första programmeraren, Ada har fått språket "Ada" uppkallat efter sig.[2]

DOM, Document Object Model

Dokument objekt modellen är ett språkoberoende och plattformsoberoende gränssnitt. Noderna av varje dokument struktureras upp i trästruktur som kallas för DOM träd. Sättet som Javascript hänger ihop med DOM är att JavaScript kan ändra html element genom att gå igenom en dokument objekt model. När man har hämtat ett element i javascript kan man "klättra" runt i html trädet för att ta sig till ett syskon eller klättra upp eller ner i trädet för att ta sig till ett annat element utan att veta dess namn eller vilket det är. 1994 stöttade World Wide Web Consortium öppen standard för World Wide Web även kallad webben. Netscape Communications och Microsoft skapade tillsammans ett språk som skulle passa för både webbläsarna som fick namnet ECMAScript. Efter att ECMAScript släppts så började W3C att standardisera DOM. Det som kallas för DOM LEVEL 1 var rekommenderande för W3C på 1998 talet. Sedan har DOM LEVEL 2 & 3 publicerats och den fjärde versionen är under utveckling.[3]
